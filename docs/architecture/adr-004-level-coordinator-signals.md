# ADR 004: Level Coordinator Signals Refactor Decision\n\n## Context\nThe LevelCoordinator currently has ~421 lines and a growing _connect_signals() function. We need to decide whether to refactor this into a data-driven signal table now or defer it.\n\n## Decision\nWe choose to **defer** the refactor in favor of accepting the current implementation for now. Wait until the file hits 500 lines or we add a 4th world context.\n\n## Rationale\nAt 421 lines, the file is close to the threshold but still manageable. The S5 critical path prioritizes polishing and export over preemptive refactoring. By re-accepting with an explicit trigger (4th world context), we save time now while maintaining technical oversight.\n
